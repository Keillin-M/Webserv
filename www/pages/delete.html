Recommended Testing Workflow

  1. Step 1: Create Test File
    Go to POST Tests and upload a test file
  2. Step 2: Verify File Exists
    Go to GET Tests and check that the file is accessible
  3. Step 3: Delete File
    Use the form above to delete the file (note the path from Step 2)
  4. Step 4: Verify Deletion
    Try to GET the file again - should return 404 Not Found

Expected Results

  • 200 OK / 204 No Content: File successfully deleted
  • 403 Forbidden: DELETE not allowed on this path
  • 404 Not Found: File doesn't exist (already deleted or never existed)
  • 405 Method Not Allowed: DELETE method not configured for this location
  • 500 Internal Server Error: Server error during deletion

Security Considerations

  • Your server should validate paths to prevent deletion of system files
  • DELETE should only be allowed in designated directories (e.g., /uploads/)
  • Consider implementing authentication for DELETE operations
  • Test that directory traversal attacks (../) are properly blocked
